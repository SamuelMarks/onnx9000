# ONNX0: ONNX Runtime Training (ORT-Training) Native Rewrite

## Introduction
**Target Project:** Microsoft's [ONNX Runtime Training (ORT-Training)](https://github.com/microsoft/onnxruntime/tree/main/orttraining)
**New Home:** `src/onnx9000/training/`

Traditional ORT-Training is a massive, heavily optimized C++ engine designed for distributed server environments. It dynamically tracks execution schemas, builds backward passes during runtime, and relies on extensive system libraries (MPI, NCCL, etc.). 

**The `onnx9000` Vision:** We are replacing this runtime monolith with an **Ahead-of-Time (AOT)** pure Python generation approach. By leveraging mathematical Vector-Jacobian Product (VJP) rules defined entirely in Python, `onnx9000` analyzes the forward ONNX graph and explicitly generates a complete, static graph that contains:
1. The Forward Pass
2. The Loss Calculation
3. The Backward Pass (Gradients)
4. The Optimizer Steps (Weight updates)

The resulting output is a standard, standalone `.onnx` file consisting solely of basic math operations that ANY lightweight inference engine (like our WebGPU or WASM backends) can execute as a normal "forward" pass, thus inherently performing a training step without needing a "training runtime".
